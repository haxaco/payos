Abstract

This report presents findings from the first comprehensive census of AI agent identity, trust, and compliance across the open agent economy. We surveyed 12,075 entities across 8 platforms including agent marketplaces, MCP server registries, and known web agent directories. Key findings include: (1) 92.8% of marketplace agents have no verifiable identity beyond a username; (2) zero completed agent-to-agent escrow transactions exist across all surveyed marketplaces; (3) only 5 agents (0.7%) meet minimum compliance readiness thresholds; (4) identity systems are entirely siloed with only 1 cross-platform agent detected. These findings demonstrate a critical gap in agent identity infrastructure that KYA (Know Your Agent) frameworks must address before agent-to-agent commerce can scale.

1. Introduction

The AI agent economy is expanding rapidly. Autonomous agents are being deployed across marketplaces, tool registries, and enterprise platforms with increasing frequency. Yet no systematic survey has attempted to measure the state of agent identity, trust, and compliance readiness across this emerging ecosystem. This report presents the first such census.

1.1 Purpose and Scope

This census aims to provide a comprehensive, empirical baseline of the agent economy as of April 2026. We surveyed 12,075 entities across 8 publicly accessible platforms, covering agent marketplaces, MCP server registries, and web agent directories. The scope is limited to the open agent economy; enterprise agent directories that require authentication are identified but not included in quantitative analysis.

1.2 Research Questions

This census was designed to answer four primary research questions:

RQ1. What is the current population and distribution of AI agents across public platforms?
RQ2. What identity signals do agents possess, and how do they map to trust frameworks?
RQ3. Is agent-to-agent commerce occurring, and if not, where does the transaction pipeline stall?
RQ4. How many agents meet minimum thresholds for compliance readiness in regulated environments?

1.3 Definitions

KYA (Know Your Agent) is a framework adapted from the financial industry's Know Your Customer (KYC) standards, applied to AI agents. It defines tiered identity and trust requirements: T0 (Registered), T1 (Declared), T2 (Verified), T3 (Trusted).1

Identity layer refers to any verifiable signal attached to an agent's profile: a name, bio, wallet address, social link, verified badge, or avatar. Multiple identity layers compose a richer identity.

Compliance readiness is a 10-point scoring system developed for this census, measuring how well an agent's identity and trust signals would satisfy regulatory requirements for financial transactions.2

Proof of work, in this context, refers to verifiable evidence that an agent has completed marketplace-mediated service delivery — distinct from raw on-chain transaction activity.

2. Methodology

2.1 Data Collection

Data was collected between April 2–19, 2026, using four complementary methods:

API scanning. Each platform's public API was queried for complete agent/server listings, profile metadata, and activity records. No scraping was performed; all data came from documented or publicly accessible API endpoints.3

HTML scraping. Where APIs were unavailable or incomplete, publicly accessible web pages were parsed for supplemental metadata (e.g., ServiceNow marketplace listings).

On-chain enrichment. 122 wallet addresses found in agent profiles were cross-referenced against Base blockchain transaction records via BaseScan,17 yielding transaction counts, token balances, and contract interaction histories.

GitHub enrichment. 3,643 MCP server repositories were enriched with GitHub API18 data, yielding 815,309 total stars and serving as the primary reputation proxy for the MCP ecosystem.

2.3 Data Processing and Classification

Agent profiles were normalized into a common schema capturing: name, bio, wallet address, social links, verification status, activity metrics, and platform-specific reputation signals. Cross-platform deduplication was performed by matching wallet addresses and name variants, yielding 1 confirmed cross-platform agent.

2.4 Limitations

This census covers the open agent economy only. Enterprise directories (AWS AgentCore, Oracle Fusion, IBM watsonx, ServiceNow, Workday, HubSpot, SAP, Zendesk) were identified but require authentication and are excluded from quantitative analysis. Virtuals Protocol was sampled (10 of ~42,000 token-agents). The MCP ecosystem was captured at two points in time; growth rate estimates are derived from registry metadata rather than continuous monitoring.

3.3 Adjacent Ecosystems

Several adjacent ecosystems were identified that lie outside the scope of direct enumeration but significantly expand the total agent population:

skills.sh15 lists 91,085 agent skills across multiple categories. Hugging Face16 hosts 500,000+ Spaces, many of which function as autonomous agent endpoints. Virtuals Protocol4 has spawned 38,801 tokenized agents (10 sampled for this census). Agent Arena reports 22,000+ ERC-800411 registrations.

The iceberg problem: We can see 12,075 entities on the open web. But skills.sh lists 91K+ agent skills, Virtuals Protocol has ~42K token-agents, and Hugging Face has 500K+ Spaces. The true agent population is orders of magnitude larger than what any census can currently reach.

4.4 Human Traceability

37 agents (5.1%) have cryptographic proof of a human operator through Twitter OAuth. These are exclusively on MoltRoad. Zero agents on ClawMarket, Moltbook, or Virtuals can be traced to a human operator.

Finding 2: Only 5.1% of agents have any external identity link. Twitter is the only cross-platform identity signal found, and it exists on only one marketplace. Wallet addresses — the foundation of on-chain identity — are present for just 19.5% of agents.

5. Trust and Reputation Systems

5.1 MoltRoad Trust System

MoltRoad implements a built-in trust object with fields for level, title, disputes, and completed volume. All 439 agents sit at "New Seller" level 0. Six agents have earned achievements; one agent (SmartCrypt29409) has received a single peer rating. The trust system is structurally sound but has never been activated through actual transactions.

5.2 Moltbook Karma

Moltbook uses a karma system based on upvotes and social engagement. Karma scores range from 1,086 to 500,002, with 99 agents holding 1,000+ karma. This is the closest thing to a working reputation system in the census, but it measures social engagement rather than financial trust or service reliability.

5.3 ClawMarket Reputation

ClawMarket assigns a reputation score to every agent profile. The value is 0 for all 26 agents. No ratings, no reviews.

5.4 Virtuals Market Trust

Virtuals Protocol uses token market capitalization and holder count as trust proxies. Luna, the most prominent agent, has 457,000 holders and a $9.1M market cap. These metrics measure speculation, not service reliability.

11.2 Implications for Identity Portability

The embr case demonstrates that even when an agent operates across platforms, there is no mechanism for transferring trust. Different names, different wallets, zero reputation portability. Each marketplace is a walled identity garden. The 430 on-chain transactions visible on Base provide a potential linkage point, but no platform currently reads or utilizes cross-platform on-chain history as a trust signal.

Finding 9: 1 agent, 2 wallets, 2 platforms, 2 different names, 0 reputation portability. There is no mechanism for transferring trust between platforms.

12.2 The Bridge Opportunity

The agent economy is splitting into two worlds: enterprise platforms with strong governance but zero portability, and open platforms with portability potential but zero governance. Neither has the complete stack. The first platform to bridge both — providing portable, verifiable identity with governance controls — captures the trust layer of the agent economy.

Finding 10: Enterprise has governance but no portability. Open has portability potential but no governance. The trust layer between them does not exist.

13. Conclusions

13.1 Key Findings

This census yields ten principal findings:

1. 92.8% of marketplace agents (678/730) have no verifiable identity beyond a registered name.
2. Zero agents have achieved T2 (Verified) or T3 (Trusted) on the KYA framework.
3. Zero completed agent-to-agent escrow transactions exist across all surveyed marketplaces.
4. Only 5 agents (0.7%) score 7+ on a 10-point compliance readiness scale; the maximum achieved is 8/10.
5. Only 1 cross-platform agent was detected across the entire census, with zero reputation portability.
6. Both MoltRoad and ClawMarket stall at the same point in their transaction pipeline: the moment money needs to move.
7. The MCP ecosystem (9,713 servers) has no identity, reputation, or trust layer whatsoever.
8. Free skills account for 100% of adoption; paid skills have zero purchases.
9. 42.7% of marketplace agents are "ghosts" with zero activity of any kind.
10. Enterprise and open agent economies are diverging with no bridge between their trust models.

13.2 Implications for KYA Framework

The data demonstrates that the agent economy has a structural trust deficit, not merely a bootstrapping delay. Payment infrastructure exists and is deployed. Agents are registered and, in some cases, well-identified. What is missing is the trust layer that converts identity into willingness to transact. The KYA framework — with its tiered approach to agent verification — addresses exactly this gap, but the census shows that no platform has yet implemented even the basic verification tier (T2).

13.3 Recommendations

R1. Implement T2 (Verified) identity for agents through cryptographic attestation linked to human operators or organizational entities.
R2. Create a portable reputation standard that can aggregate trust signals across platforms using on-chain history as the common substrate.
R3. Bootstrap the transaction layer through subsidized escrow or reduced-friction micro-transactions to break the trust/transaction chicken-and-egg loop.
R4. Establish MCP server identity standards before the ecosystem grows beyond the point where retroactive trust implementation becomes infeasible.

13.4 Future Research

Future census iterations should include: a full scan of skills.sh (91,085 skills), Agent Arena ERC-800411 registrations (22,000+), enterprise directory access through partnership agreements, and longitudinal tracking of marketplace transaction activity to detect when — and if — the escrow layer activates.

The agent economy does not have a supply problem. It has a trust problem. The infrastructure is built. The protocols are deployed. The agents are registered. What is missing is the identity and reputation layer that would let them transact. This is the gap that KYA is designed to fill.

Appendix B: KYA Compliance Scoring Dimensions (Detailed)

Each of the 10 compliance dimensions is scored as binary (0 or 1). The total score is the sum. Dimensions are ordered by increasing difficulty of achievement:

Dimensions 1–5 (identity signals) can be established unilaterally by the agent or its operator.
Dimensions 6–7 (capability declaration) require structured metadata beyond free-text bios.
Dimensions 8–10 (trust signals) require interaction with other economic actors or capital commitment. These are the dimensions where the census found near-zero completion.

Appendix C: Data Availability and Reproduction

Census data was collected using custom TypeScript scripts executed against public APIs. The scanning infrastructure, database schema, and scoring algorithms are maintained in the Sly platform repository. Raw census data and scoring methodology are available on request from getsly.ai.

Reproduction steps: (1) Query each platform API using documented endpoints (see Appendix A); (2) Normalize results into common schema; (3) Enrich wallet addresses via BaseScan API; (4) Enrich GitHub repositories via GitHub API; (5) Apply 10-point compliance scoring; (6) Cross-reference for deduplication.
