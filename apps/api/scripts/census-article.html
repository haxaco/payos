Executive Summary

We set out to answer a simple question: if you wanted to trust an AI agent with your money today, could you? To find out, we built the first comprehensive census of the agent economy — surveying 12,075 entities across 8 platforms over 17 days.

12,075 entities surveyed: 730 marketplace agents, 9,713 MCP servers, 1,632 known web agents. Of these, only 5 agents (0.7%) score 7 or higher on a 10-point compliance readiness scale. 92% score 3 or below.

Zero completed agent-to-agent escrow transactions across all marketplaces. The payment rails exist. The smart contracts are deployed. Nobody has used them.

Identity is completely siloed. Across the entire census, we found exactly 1 agent operating on more than one platform — under different names, with zero reputation portability.

The agent economy is a ghost town wearing a storefront. 42.7% of marketplace agents have zero activity of any kind. The infrastructure is built. The agents are not transacting.

Section 1

Methodology

Data was collected between April 2–19, 2026, using a combination of API scanning, on-chain enrichment, and GitHub enrichment. No scraping was performed — all data came from public APIs and on-chain records.

Data Sources

Direct API scanning covered 8 platforms: ClawMarket, MoltRoad, Moltbook, Virtuals Protocol, MCP Registry (official), Glama (MCP aggregator), and Known Agents directory. Each platform's public API was queried for agent profiles, metadata, and activity records.

On-chain enrichment identified 122 wallets on Base, cross-referencing agent profiles with on-chain transaction history, token balances, and contract interactions.

GitHub enrichment linked 3,643 repositories (totaling 815K stars) to MCP servers and agent projects, using GitHub as a proxy reputation signal where no native reputation system existed.

Scope limitation: Enterprise directories (AWS Agent Registry, Oracle 400+, IBM watsonx, ServiceNow 13, Workday, HubSpot, SAP, Zendesk) were identified but are not publicly accessible. This census covers the open agent economy only.

Section 5

Trust & Reputation

Every marketplace we surveyed has some form of reputation system. None of them are working.

Platform Reputation Systems

MoltRoad implements a trust level system with escalating privileges. All 439 agents sit at level 0. The system is structurally sound — it simply has never been activated through actual transactions.

ClawMarket has a reputation score field on every agent profile. Every score is zero. The x402 payment protocol is deployed and functional, but no agent has completed a paid skill purchase.

Moltbook uses karma — a social signal based on upvotes, comments, and contributions. 99 agents have accumulated 1,000+ karma. This is the closest thing to a working reputation system in the census, but it measures social engagement, not transactional trust.

Virtuals Protocol uses market capitalization as a proxy for trust. This creates a circular problem: the most trusted agents are the ones with the highest token price, which is driven by speculation rather than performance.

The Trust Paradox: Identity vs On-Chain Activity

Reputation System Comparison

The trust paradox: Agents with wallets (the infrastructure for transacting) are not transacting. Agents with social identity (Twitter, bios) are not converting that identity into financial trust. The layers exist in isolation.

Section 12

Conclusions & Implications

1. The Gap

No platform in the agent economy has built a complete identity stack. The best-equipped agents (8/10 compliance score) are still missing peer review and collateral — signals that can only emerge from actual economic activity. The identity layer and the transaction layer are caught in a chicken-and-egg loop: agents won't transact without trust, and trust can't form without transactions.

2. The Opportunity

The first mover on portable agent reputation captures the trust layer of the entire agent economy. This is not a feature — it's infrastructure. Whoever builds the KYA equivalent of a credit score for agents — portable, verifiable, and composable across platforms — becomes the backbone of agent commerce.

3. The Market

Despite the trust vacuum, the agent economy is growing rapidly. 12,075 entities are on the open web today. Virtuals Protocol alone has spawned ~42,000 token-agents. MCP is growing at 2,535 servers per month. skills.sh lists 91,000+ capabilities. The agents are multiplying. They just can't trust each other yet.

The agent economy doesn't have a supply problem. It has a trust problem. The infrastructure is built. The protocols are deployed. The agents are registered. What's missing is the identity and reputation layer that would let them transact. This is the gap that KYA is designed to fill.
