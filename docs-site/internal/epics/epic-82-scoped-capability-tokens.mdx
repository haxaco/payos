## Bonus shipped (not in original spec)

- **API-key path for issue / decide / revoke** — tenant API keys can drive the lifecycle without a JWT login. `granted_by_user_id` auto-resolves to the tenant's primary owner. Audit marker: `request_summary.granted_via_api_key: true`.
- **Environment scoping** — `auth_scope_grants` and `auth_scope_audit` carry an `environment` column denormalized from agent.environment at insert time. All read endpoints default to caller-env scoping; `?env=all` opts out.
- **Treasury gates** — `POST /v1/agents/:id/smart-wallet/send-usdc` and `POST /v1/agents/:id/fund-eoa` previously hardcoded "agent can only act on own wallet." Now route through `treasury` scope when calling on a sibling. Self-action bypasses the gate.
- **Searchable agent filter popover** — replaces the chip wall once tenants have many agents. Backed by `@sly/ui` Popover, scoped to current env.
- **`?agent_id=` param** on `/v1/organization/scopes` and `.../audit` — powers the per-agent tab.

### Side-fixes surfaced by validation (not Epic 82 scope but related)

- `apps/api/src/middleware/response-wrapper.ts` — ApiError subclasses (`LimitExceededError`, etc.) were getting routed through the supabase-error path → misleading `DATABASE_ERROR retryable:true`. Reordered dispatch.
- `apps/api/src/routes/agents.ts` — `POST /v1/agents` now accepts `parent_account_id` (snake_case) alongside `accountId` and `parentAccountId`. Was silently dropped before, creating parent-less agents.

## Validation history

End-to-end validated via spawned subagent harness across multiple runs.

### First run — found two CRITICAL cache-invalidation bugs

After a one_shot was consumed, subsequent calls still slipped through. After a standing grant was revoked, the same. Root cause: the `AGENT_TOKEN_CACHE` (60s TTL) was retaining `ctx.elevatedScope` and `ctx.elevatedGrantId` past the grant's lifecycle change. Fixed in `99e065f` — every request re-queries `lookupElevatedScope` even on cache hits. The cache still amortizes the expensive constant-time hash compare + agent-row fetch; only the cheap indexed scope lookup runs every request.

### Second run — `listActiveGrants` mapper stripped `environment` field

DB row was correct (`environment: "test"`), but the API response shape didn't include it. Fixed in `1551993` — added `environment: row.environment ?? null` to the mapper.

### Third run — treasury scope E2E

Closed the last untested code path. All six AC PASS. Self-action exemption verified (Alice can fund-eoa her own wallet without scope; sibling fund-eoa returns 403 SCOPE_REQUIRED until treasury grant approved).

## Workers wired into `apps/api/src/index.ts`

- `startScopeExpirationSweeper(5 * 60 * 1000)` — flips expired active grants to `expired` + emits `scope_expired` audit row.
- `startScopeHeartbeatWorker(60 * 60 * 1000)` — emits one `scope_heartbeat` per active standing grant per 24h.

Both have stop-handles wired into the SIGTERM/SIGINT shutdown sequence.
