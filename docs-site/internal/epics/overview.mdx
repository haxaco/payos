## Why this page is thin

Epic docs change frequently and belong in the PRD tree, not mirrored here. The authoritative source is `docs/prd/epics/`. This internal portal page exists so the nav has an entry; treat it as a pointer, not content.

## Adding an entry

If an epic has implementation notes worth persisting across the epic's lifetime, create a dedicated page under `internal/protocol-internals/` or `internal/engineering/` and link from the epic doc. Epics themselves should stay in the PRD tree.
