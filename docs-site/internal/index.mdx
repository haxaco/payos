<Warning>
**Internal documentation.** This portal contains architecture details, runbooks, and security controls not meant for external distribution. Access is limited to Sly staff via Google Workspace SSO.
</Warning>

This is the staff-facing documentation portal. It mirrors the [public docs](https://docs.getsly.ai) and adds:

- **Internal API reference** — admin routes, dashboard-internal endpoints, and everything not surfaced externally
- **Architecture** — how the monorepo is organized, where code lives, why
- **Runbooks** — deploys, incidents, on-call procedures
- **Protocol internals** — implementation notes, gotchas, and historical context
- **Scanner** — the demand intelligence service (not in public docs yet)
- **Compliance & security** — KYC/KYB flows, sanctions screening, data retention

## Content ownership

Code documentation is owned by the team that owns the code:

- **`apps/api`** → platform team
- **`apps/web`** → frontend team
- **`apps/scanner`** → growth team
- **`packages/*`** → platform team
- **Runbooks** → SRE / on-call rotation
- **Compliance & security** → compliance officer

PRs that modify routes must update the narrative docs in the same PR. Schema changes propagate automatically via `@hono/zod-openapi`, so the API reference stays in sync without manual work.
