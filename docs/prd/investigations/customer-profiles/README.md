## Profile Structure

Each profile document contains:

1. **Description** — What the business does
2. **Business Model** — How they make money
3. **Transaction Profile** — Volume, size, frequency patterns
4. **Corridors & Currencies** — Geographic focus and currency pairs
5. **Pain Points** — 5 specific problems they face
6. **PayOS Integration** — API flow examples
7. **Test Scenarios** — 4 scenarios with expected outcomes
8. **Seed Data Requirements** — What data to generate for testing

## Usage Guide

### For Development Testing
```
1. Select profile(s) relevant to feature being tested
2. Use profile name as `partner_type` in seed data
3. Generate seed data per requirements
4. Execute test scenarios
5. Validate expected outcomes
```

### For Demos
```
1. Identify prospect's industry/segment
2. Select matching category and profile
3. Configure demo environment with profile data
4. Walk through relevant test scenarios
5. Show protocol-specific capabilities
```

### For Sales/Marketing
```
1. Use pain points to craft messaging
2. Reference transaction volumes for sizing
3. Map features to specific use cases
4. Build ROI models from profile economics
```
