How Readiness Scores Work

Each company receives a composite Readiness Score (0–100) computed as a weighted average of four component scores. Here's what each measures and how to interpret the grades:

Protocol Score (40% weight)

Measures agentic commerce protocol support
  • UCP — 30 pts confirmed (Universal Checkout Protocol)
  • ACP — 20 pts confirmed (Agent Checkout Protocol)
  • MCP — 15 pts confirmed (Model Context Protocol)
  • x402 / AP2 — 10 pts each confirmed
  • NLWeb / Visa VIC / Mastercard — 5 pts each
  • Status multipliers: confirmed 1.0x, platform_enabled 0.5x, eligible 0.3x

Data Score (25% weight)

Evaluates structured data quality for agent parsing
  • JSON-LD present — +25 pts
  • Schema Product — +20 pts, Schema Offer — +15 pts
  • OpenGraph metadata — +10 pts
  • Product attribute completeness (price, availability, SKU, image) — up to +30 pts

Accessibility Score (20% weight)

How bot-friendly is the site? (starts at 100, deducts)
  • Blocks all bots — -40 pts
  • Has CAPTCHA — -25 pts
  • Requires JavaScript — -15 pts
  • Blocks GPTBot/ClaudeBot — -10 pts each
  • Explicitly allows agents — +10 pts bonus

Checkout Score (15% weight)

Payment friction and method diversity
  • Guest checkout available — +30 pts
  • No account required — +20 pts
  • Low checkout steps (1–3) — +15 to +25 pts
  • 3+ payment processors — +10 pts
  • Crypto +5, Pix +3, SPEI +2

Cloudflare Web Bot Auth — Agent Identity Standard (live, growing adoption)

Cloudflare's Web Bot Auth protocol lets AI agents prove their identity using cryptographic signatures (RFC 9421). Agents publish public keys at /.well-known/http-message-signatures-directory. Both Visa TAP and Mastercard AgentPay build on the same cryptographic foundation.

Business case: Web Bot Auth replaces IP allowlists and CAPTCHA with cryptographic agent identity. Merchants using Cloudflare can automatically permit verified agents from Visa, Mastercard, or any registered AI agent — while keeping malicious bots out. This is the identity layer that all agentic commerce protocols need.
